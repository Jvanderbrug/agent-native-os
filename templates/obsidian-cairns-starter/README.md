# Obsidian Cairns Starter Vault

This is the smallest useful local second-brain structure for the workshop.

It is not production Cairns. It is an Obsidian-friendly vault structure that lets Claude Code:

- Save raw scratchpad captures
- Log decisions
- Keep short L1 waypoints always available
- Write L2 Chain-of-Density Card Catalog notes
- Build toward a morning brief

## Recommended setup

Run `/personalize` first. At the end, answer yes to:

```text
Scaffold your personal Cairns vault now? [Y/n]
```

That path invokes the `cairns-init` skill, writes a local `CLAUDE.md`, seeds the day-one L1 waypoint files, and leaves remote vector and graph systems off by default.

## Manual install

From the repo root:

```bash
mkdir -p "${HOME}/Documents/second-brain"
cp -R templates/obsidian-cairns-starter/. "${HOME}/Documents/second-brain/"
if [ ! -f "${HOME}/Documents/second-brain/.gitignore" ]; then
  mv "${HOME}/Documents/second-brain/gitignore.template" "${HOME}/Documents/second-brain/.gitignore"
else
  rm "${HOME}/Documents/second-brain/gitignore.template"
fi
```

Then install the commands and skills:

```bash
mkdir -p "${HOME}/.claude/commands" "${HOME}/.claude/skills"
cp configs/commands/log-decision.md "${HOME}/.claude/commands/log-decision.md"
cp configs/commands/personalize.md "${HOME}/.claude/commands/personalize.md"
cp -R configs/skills/second-brain-capture "${HOME}/.claude/skills/second-brain-capture"
cp -R configs/skills/cairns-init "${HOME}/.claude/skills/cairns-init"
cp -R configs/skills/cairns-ingest "${HOME}/.claude/skills/cairns-ingest"
cp -R configs/skills/cairns-query "${HOME}/.claude/skills/cairns-query"
cp -R configs/skills/cairns-lint "${HOME}/.claude/skills/cairns-lint"
```

Restart Claude Code after installing.

## Folder map

```text
CLAUDE.md                    Personalized profile from /personalize
AGENTS.md                    Layer contract for agents working inside the vault
inbox/scratchpad/            Raw things dropped in before processing
decisions/                   Full decision records from /log-decision
cairns/L1/INDEX.md           Master waypoint table
cairns/L1/personal/          Day-one personal waypoints
cairns/L2/cards/             Chain-of-Density cards, one per source
cairns/L2/decisions/         Decision cards from /log-decision
cairns/L3/articles/          Article and URL captures
cairns/L3/notes/             Notes and scratchpad captures
cairns/L3/transcripts/       Meeting, voice, and video transcripts
cairns/L3/decisions/         Raw decision context
daily-briefs/                Morning brief outputs
meeting-prep/                Meeting prep outputs
daily-reviews/               End-of-day outputs
```

## Retrieval contract

When asking questions from this vault, agents should use:

```text
L1 route map -> L2 Card Catalog -> L3 raw evidence
```

L1 is selective and human-approved. L2 is generated by Chain-of-Density loops and links back to raw sources. L3 is immutable evidence.

## 4D and 8D paths

4D is the default. It is markdown-only and skips Supabase, Neo4j, Docker, and remote services.

8D is optional. It may add Supabase pgvector and Neo4j later, but those stores must use isolated personal namespaces and must never collide with the AI YouTube demo or production Cairns labels.
